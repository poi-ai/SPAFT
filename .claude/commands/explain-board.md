# 板情報の構造説明

KabuStation API の板情報レスポンス構造と、SPAFTでの利用方法を説明する。

---

`src/kabusapi/info.py` と `sql/ddl/board.sql`（またはそれに相当するDDL）を読んだうえで、以下を説明すること。

## 説明項目

### 1. `/board/{証券コード}@{市場コード}` レスポンスの主要フィールド

各フィールドの名前、型、取引判断への使い方を説明すること。特に重要なフィールド:
- `CurrentPrice` / `CurrentPriceStatus` — 現在値とそのステータス
- `BidPrice` / `AskPrice` — 最良買値・最良売値
- `BidSign` / `AskSign` — 気配区分
- `BidQty` / `AskQty` — 最良買残・最良売残
- `Sell1`〜`Sell10` / `Buy1`〜`Buy10` — 板情報（価格・数量・気配フラグ）
- `OverSellQty` / `UnderBuyQty` — OVER/UNDER数量
- `TradingVolume` / `TradingValue` — 売買高・売買代金

### 2. BidSign / AskSign の値と意味

| 値 | 意味 | 注文時の注意 |
|---|---|---|
| `0101` | 一般気配 | 通常の指値注文が存在する状態 |
| `0102` | 特別気配 | 需給不均衡。この状態での注文は注意が必要 |
| `0103` | 注意気配 | |
| `0107` | 寄前気配 | 寄付き前の気配 |
| `0108` | 停止前特別気配 | |
| `0109` | 引け後気配 | |
| `0116` | 寄前気配（買/売で異なる） | |

特別気配（`0102`）の場合、ストップ高/安の可能性があり、注文を出しても約定しないことがある。
注文前に `BidSign` / `AskSign` を確認することが推奨される。

### 3. CurrentPriceStatus の値と意味

通常の連続取引時は `1` になる。以下の値の場合は取引状態に注意:

| 値 | 意味 |
|---|---|
| `1` | 現値（通常） |
| `2` | 前日終値 |
| `3` | 特別気配中 |
| `4` | 一時中断/停止中 |
| `5` | 対象外 |
| `7` | 気配なし |

### 4. OVER / UNDER の意味

- `OverSellQty`: 表示されている売り板10本より上に積み上がっている売り数量の合計
- `UnderBuyQty`: 表示されている買い板10本より下に積み上がっている買い数量の合計

これらが大きいと「板が厚い（流動性が高い）」状態を示す場合があるが、
逆に極端に大きい場合はアルゴリズム注文等による特殊な板状況の可能性もある。

### 5. SPAFTのboardsテーブルとの対応

DBに保存されているカラムとAPIレスポンスのフィールド名の対応を説明すること。
ML学習データとして `boards` テーブルを使う際の注意点（データの意味、欠損値の扱い等）も記載すること。

### 6. 注文前の板チェックのベストプラクティス

注文を出す前に確認すべき板の状態（最良気配の確認、特別気配の回避、OVER/UNDERの読み方）を
実際のコードパターンを例示しながら説明すること。
