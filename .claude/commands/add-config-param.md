# 設定パラメータの追加

引数で指定された設定パラメータを `config.py.sample` に追加し、使用箇所を提示する。

パラメータ名と目的: $ARGUMENTS

---

## 手順

### 1. `src/config.py.sample` を読む

既存のセクション構成（スキャルピング関連、板情報記録関連、四本値関連）を確認する。

### 2. 適切なセクションに追加、またはセクションを新設

追加形式:
```python
# <パラメータの目的を日本語で説明>（有効な値の例や範囲も記載）
<PARAM_NAME> = <デフォルト値>
```

**規約:**
- コメントは日本語で書く
- デフォルト値は実際に使える値を設定する（`None` は避ける）
- フラグ値（True/False）の場合はデフォルトを安全な側（機能OFF等）にする
- 数値パラメータの場合は単位を明記する（例: 「pip数」「秒」「円」）
- `config.py` は **絶対に変更しない**（gitignore対象）

### 3. 使用箇所の特定

以下のコマンドで関連ファイルを検索し、新しいパラメータを使うべき場所を特定する:
- 関連する機能のサービスクラス
- 関連する起動ファイル（Controller）

### 4. 以下を出力する

1. `config.py.sample` への追加内容（差分）
2. 使用すべき箇所のファイル名と行番号
3. 各箇所でどのように使うか（`import config` して `config.<PARAM_NAME>` でアクセス）

**注意:** このコマンドは `config.py.sample` の変更提案のみ行う。
実際のロジック変更は別途実装すること。
