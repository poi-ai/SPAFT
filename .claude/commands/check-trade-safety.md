# トレード安全性チェック

引数で指定されたファイルまたはコードをレビューし、SPAFT固有のリスクがないか検査する。

対象: $ARGUMENTS

---

以下の7項目を順番にチェックし、各項目について「問題なし」または「問題あり（ファイル名:行番号、内容、修正案）」を報告すること。

## チェック項目

### 1. APIレート制限違反
- 発注系（sendorder, cancelorder 等）のAPI呼び出しが **5回/秒** を超える可能性がないか
- 余力・銘柄情報系のAPI呼び出しが **10回/秒** を超える可能性がないか
- ループ内でAPIを呼ぶ処理に `time.sleep()` がないケースを特にチェック

### 2. 市場時間ガードの欠如
- 注文を発行するコードが `exchange_time()` の結果を確認しているか
- お昼休み（戻り値 `4`）や大引け後（戻り値 `5`）にも注文を送れてしまう状態になっていないか
- `config.AM_END_TIME` / `config.END_TIME` との整合性

### 3. 呼値を無視した価格計算
- `price + 1` / `price - 1` / `price + N` のような生の加減算が含まれていないか
- 価格計算には `self.util.stock_price.get_updown_price()` または `get_price_range()` を使うこと
- `ORDER_LINE`, `SECURING_BENEFIT_BORDER`, `LOSS_CUT_BORDER`, `TRAIL` はpip数（≠円）であることを考慮しているか

### 4. WebSocket昼休み切断対応
- WebSocketを使うコードで昼休み（11:30〜12:30）の切断を検知・再接続する処理があるか
- 前場終了前（11:30前）にWebSocketを切断し、後場開始前（12:29:59）に再接続するフローになっているか

### 5. 設定値のハードコード
- `config.py` から取得すべき値（パスワード、証券コード、パラメータ等）がソースコード内に直書きされていないか
- APIトークン、パスワード、LINE TOKENが文字列リテラルとして書かれていないか

### 6. 戻り値のタプル規約
- Serviceクラスの公開メソッドが `(bool, value_or_error)` のタプルを返しているか
- 呼び出し側で戻り値の `bool` を確認してからデータを使っているか

### 7. 呼び出し階層の違反
- Controllerファイル（`src/*.py`）が `self.api` や `self.db` を直接呼んでいないか
  - Controllerは `self.service.xxx` 経由で呼ぶのが正しい
- Serviceクラスが `KabusApi()` や `Db()` を自分でインスタンス化していないか
  - `ServiceBase.__init__()` で生成された `self.api`, `self.db` を使うこと

---

各問題について以下の形式で報告すること:
```
[項目名] ファイル名:行番号
問題: <何が問題か>
修正案: <どう直すか>
```

問題がなければ「✅ 全項目問題なし」と報告すること。
